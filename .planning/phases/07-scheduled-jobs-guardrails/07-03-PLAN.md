<objective>
Wire the Phase 7 modules into the CLI (`pkm lint`, `pkm dashboard`, `pkm backfill-embeds`) and add the reusable `backfill_embeds` function to pkm/retrieval/embed.py.

Purpose: The nightly workflow (Plan 04) calls these CLI subcommands. `backfill-embeds` is an EXPLICIT SCOPE ADDITION closing the deferred Wave 3 gap (STATE.md / PROGRESS.md Phase 6): CI ingest currently skips the embed step because CF_ACCOUNT_ID/CF_API_TOKEN are not yet GitHub Actions secrets, so claims ingested in CI lack Vectorize embeddings and the query worker goes stale. `backfill-embeds` is the reusable replacement for the throwaway Wave 3 backfill script — it embeds every claim lacking an embeddings_meta row, idempotently. It is NOT a GUARD requirement; it belongs in Phase 7 because Phase 7 owns the scheduled jobs surface and this is the maintenance command the nightly job (or operator) uses to keep Vectorize complete after creds are added.
Output: pkm/retrieval/embed.py (backfill_embeds), pkm/cli.py (3 new subcommands), tests/test_backfill_embeds.py.
</objective>
