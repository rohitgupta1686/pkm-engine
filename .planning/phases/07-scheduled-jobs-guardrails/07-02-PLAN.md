| DB writes → dashboard_counters | Counter bumps happen inside ingest transaction; a bump failure would abort the parent transaction |
| dashboard output → vault | dashboard.md committed back to private vault repo via existing workflow step |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | counter bump SQL | mitigate | bump_counter uses parameterized ? placeholders + INSERT ON CONFLICT UPDATE; no f-string value interpolation (T-03-03) |
| T-07-02-02 | Repudiation | counter updates | accept | dashboard_counters.updated_at records last-bump time; log.md still holds the append-only audit trail |
| T-07-02-03 | Information disclosure | dashboard.md | accept | dashboard.md contains only aggregate counts + orphan list (source titles/slugs, not secrets); committed to private vault repo |
| T-07-02-04 | Elevation of privilege | counter bump on re-run | mitigate | upsert_source/upsert_concept guard bump on created=True only; insert_claim always bumps (claims are always new) — test asserts idempotent re-ingest leaves counters stable |
