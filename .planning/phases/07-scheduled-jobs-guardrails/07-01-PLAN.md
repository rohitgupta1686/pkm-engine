  <action>
    Create pkm/lint.py defining a LintReport dataclass with fields broken_wikilinks: list[dict], orphans: list[str], missing_provenance: list[dict] and a property is_clean (all three empty). Define lint_vault(conn, vault_root: pathlib.Path, write_log: bool = True, now: datetime | None = None) -> LintReport.

    Broken-wikilink check: walk wiki/sources/*.md and wiki/concepts/*.md; for each file, find all [[token]] occurrences via a regex (reuse the [[...]] form emitted by vault.py — capture the inner slug token, strip whitespace). Build the set of existing wiki page slugs from the filenames under wiki/sources/ and wiki/concepts/ (stem of each .md). A [[token]] is broken if no page exists whose stem equals the token. Record {page: vault-relative path, link: token}.

    Orphan check: a wiki page (sources + concepts) is an orphan if (a) its slug is not referenced by any [[slug]] token in any other wiki page AND (b) its slug does not appear in index.md (read vault_root/index.md if present). Skip index.md and log.md and dashboard.md themselves. Record the vault-relative path of each orphan.

    Missing-provenance check: run parameterized SQL "SELECT id, statement, source_id FROM claims WHERE chunk_id IS NULL" against the passed conn. Record {claim_id, statement, source_id} per row. Do NOT use string interpolation for values.

    Log writer: when write_log and not report.is_clean, build a single multi-line string starting with "{now_iso} lint FAIL broken={n1} orphan={n2} missing_provenance={n3}\n" followed by up to N (default 50) detail lines, then call pkm.store.vault.append_log(vault_root, block). When write_log and report.is_clean, append a single "{now_iso} lint ok\n" line (so the audit log shows the nightly run happened). Use datetime.now(timezone.utc) formatted "%Y-%m-%dT%H:%M:%SZ" when now is None.

    Pure function: no Settings, no global state, no network. conn and vault_root are arguments.
