    using abc.abstractmethod on @classmethod or as class-level typing.ClassVar annotations — use
    typing.ClassVar so subclasses set them as plain class attributes without decorators:

      role: ClassVar[str]               — agent name key, used as agent_name in LLMClient.call()
      model: ClassVar[str]              — model string (HAIKU, SONNET, or OPUS from pkm.llm.models)
      prompt_template: ClassVar[str]    — filename of the prompt (e.g. "summarize.v1.md")
      prompt_version: ClassVar[str]     — version string for cache key (e.g. "v1")
      input_schema: ClassVar[type]      — pydantic BaseModel or None (used for documentation only)
      output_schema: ClassVar[type[BaseModel]]  — pydantic model class passed to LLMClient.call()
      memory_tier: ClassVar[str]        — "stateless" | "working" | "long"

    Implement __init_subclass__ to enforce that concrete subclasses (those not themselves abstract)
    declare all seven ClassVar attributes. Raise TypeError at class definition time if any are missing.
    Use `hasattr(cls, attr)` check inside __init_subclass__.

    Implement _load_prompt(self) -> str: reads the file at
    Path(__file__).parent.parent / "prompts" / self.prompt_template and returns its content.

    Implement run(self, llm_client, input_text: str, source_id: str | None = None,
                  extra_messages: list[dict] | None = None) -> BaseModel:
      1. Load the prompt via _load_prompt().
      2. Build messages: [{"role": "user", "content": f"{prompt}\n\n{input_text}"}] plus any extra_messages.
      3. Call llm_client.call(agent_name=self.role, model=self.model, prompt_version=self.prompt_version,
         messages=messages, input_text=input_text, source_id=source_id, output_schema=self.output_schema).
      4. If result["cached"] is True, raise a RuntimeError("Cache hit — caller must retrieve from agent_runs")
         so the pipeline knows not to process a cached result as fresh output.
      5. Return result["result"] (the validated pydantic instance).

    Do NOT implement repair-retry inside BaseAgent — LLMClient._extract_result already does one
    repair-retry. BaseAgent.run() propagates any ValidationError from a second failure.
